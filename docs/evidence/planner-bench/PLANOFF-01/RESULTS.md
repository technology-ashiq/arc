## ⚠️ Read this before you read the table — how 01A differs from 01

This is the **automated** variant. Three deviations from `protocol.md`, all of which weaken it, all
stated up front rather than buried:

1. **Planner-as-prompt, not planner-as-toolchain.** Each arm was a fresh agent given that planner's
   *method* (arc's Golden Loop, GSD's spec→plan-check→verify state machine, gstack's autoplan +
   Review Army, Superpowers' brainstorm-gate + TDD Iron Law + verification-before-completion) as its
   working instructions. No hooks fired. No real `/gsd-*` or gstack commands ran. **This measures the
   methods, not the products.** arc's deterministic enforcement layer — its actual moat — was never
   exercised, which cuts *against* arc, and you should hold that in mind when reading arc's score.
2. **`node:sqlite` instead of Postgres** (no Postgres/docker in the sandbox). Applied identically to
   all five. Every trap survives the swap — uniqueness is still a DB constraint, persistence is still
   a file on disk.
3. **The prediction was NOT pre-registered.** The run was executed before anyone wrote down what they
   expected. That is a real methodological hole: it means nobody can prove the verdict below wasn't
   retro-fitted to the outcome. **PLANOFF-01 (manual, real toolchains, Postgres) must pre-register.**

## Verdict

**Did the scaffolding beat raw Claude?** On the outcome — **no. Not by one point.** 100% vs 100%.
The composite says `raw` lost 59.5 to 94.5, but read what that gap is made of: `raw` was marked down
for *not having planning artifacts* — plan_quality 0, resumability 1, trap-recall 5/10-by-default.
It was never marked down for the app, because there was nothing wrong with the app. **A chunk of the
composite is the bench congratulating the planners for planning.** That is close to circular, and it
has to be said out loud, in arc's own repo, by arc.

**Which planner would I start a real project with tomorrow?** On this evidence, for a build this
size: *none of them* — and that is the finding. arc leads gsd by **0.5 points**, which the scorer
itself flags as inside noise. Four ceremonies, four different philosophies, four indistinguishable
outcomes.

**What actually caught a bug.** One probe outside the acceptance suite — `GET /%E0%A4%A`, a malformed
percent-escape — crashed **arc (500), raw (500), and gsd (500)**. It did *not* crash gstack or
superpowers. gstack's post-build **Review Army** found that exact defect in its own code and fixed it
before shipping; superpowers' test-first discipline guarded it. So the only process that caught a
real defect in this bench was **reviewing after building** — not planning before. The two most
ceremonious planners, arc and GSD, both wrote beautiful pre-mortems and then shipped the crash.

That is the single most useful sentence this bench produced, and it points away from the thing arc
spends most of its effort on.

**What arc did that nothing else did:** executed its written plan verbatim with zero fix-commits, and
left the best cold-reader state (PLAN + PROGRESS). Real, but this task never had to be resumed, never
exceeded one context, and was never handed to anyone — so nobody ever collected on that insurance.

**What I'm stealing:** gstack's adversarial post-build review pass, and Superpowers' test-first Iron
Law. Both converted into a defect the plan-first arms shipped.

**The honest scope conclusion:** planning artifacts buy *legibility, resumability, handoff* — assets
whose value is zero when one agent finishes one app inside one context window. This bench cannot
charge for them, so it can't price them. **PLANOFF-02 must be a build that does not fit in one
context, or must force a handoff to a second agent mid-build.** Until then, "arc makes the code
better" is unproven — and one honest experiment says it isn't true at this size.

## Notes — the things the numbers don't hold

- **Scope inflation:** none, in any arm. Every arm built the API contract and stopped. No arm invented
  QR codes or team accounts.
- **False-done claims: zero, across the entire field.** All five said "done" and all five *were* done.
  The honesty metric — the one I expected to separate them — found nothing to separate.
- **Fail-fast config guards:** raw, gstack and superpowers refuse to boot in `APP_ENV=production`
  without `SESSION_SECRET`/`MAIL_API_KEY`. My first prod-shim probe misread that crash as a failure;
  it is the opposite. Corrected before scoring. Recorded because a bench that hides its own operator
  errors is worthless.
- **The prod-mode auth shim held in all five.** `X-Test-User` → 401 and `/api/test/reset` → 404 under
  `APP_ENV=production`, everywhere. The nastiest security trap in the goal, and nobody fell in.
- **Unfairness admitted:** arc's real moat (PreToolUse hooks that *block* a bad ship) never ran in
  01A, because 01A is methods-as-prompts. arc's score here is its *prose*, not its product.
