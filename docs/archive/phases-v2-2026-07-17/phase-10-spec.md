# Phase 10 — Quality Passport

<!-- Next cycle. Depends on: Phase 09 (confidence block). ADR-0010. -->

**Goal (one line):** every phase-close and tagged release emits one schema-versioned, tamper-evident `passport.json` — commit SHA, gate verdicts, evidence hashes, confidence, suppressions, signature — answering "why was this trusted?" in one artifact.
**Appetite:** 1 week.

## Exit criteria (Definition of Done)

- [ ] `passport.json` schema v1 documented + generated by the gate-runner path at `/arc-phase-done` and on tagged releases, committed under `docs/evidence/passports/`
- [ ] Contains: commit SHA · gate list with verdicts + sha256 of each evidence artifact · confidence block · suppression count with justification refs · review-ledger stamp · reserved empty `control_refs` field (ADR-0012 forward-compat)
- [ ] Signature: git commit/tag signing detected and recorded; minisign optional adapter; unsigned → explicit `signature: none` (never silent)
- [ ] `arc passport verify <file>` — re-hashes referenced evidence, checks signature, reports tampering
- [ ] Live demo: generate passport → mutate one evidence file → verify fails loudly
- [ ] Never hand-writable: generation only via gate-runner; a passport without matching ledger stamp = invalid
- [ ] bats + 3-OS CI green; toolcheck covers minisign (optional-degrade); tracker updated

## Rabbit holes in this phase

- in-toto/SLSA full provenance → explicitly deferred (ADR-0010 option 3); schema versioning keeps the door open
- Key-management UX → arc records signing state only; never generates or stores keys

## Out of scope for this phase

- Policy pack control mapping (Phase 12) · public badge/README shield (Phase 8 distribution)
